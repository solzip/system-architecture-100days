## Day 1 — SOLID 원칙

### 오늘 배운 것
SOLID는 좋은 객체지향 설계를 위한 5가지 원칙이다.
각 원칙은 독립적이지 않고 서로 연결되어 있으며,
특히 DIP는 OCP를 달성하기 위한 수단이 된다.

### 왜 중요한가
마이크로서비스에서 각 서비스가 SRP를 지키지 않으면
서비스 경계가 모호해져 나중에 분리가 어려워진다.
DIP는 테스트 코드 작성(Mock 주입)에 직접적으로 필요하다.
